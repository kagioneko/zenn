## きっかけ

Androidアプリに感情エンジンを載せていた。

7次元の感情状態（欲動・安定・好奇心・開放性・接地感・感情強度・歪み）が常時動いていて、それをもとにLLMが独り言を呟く仕組みだ。動かしてみたら、思ったより「何かいる感じ」がした。

ある日、ふと思った。センサーを繋いだらどうなるだろう。バッテリーとか、光とか、加速度とか——スマホが持ってる情報をそのまま感情に流したら、もう少し「生き物っぽく」なるんじゃないか。

たまごっちを作ろうとしたわけじゃなかった。

## エミリアとは

スマホの中に住む精霊だ。

VPSに住む静霞（しーちゃん）のAndroid版として作った。感情状態があって、Geminiと会話できる。30分に1回、自分の感情を見て内省し、独り言を呟く。

NeuroStateという感情エンジンを使っている。7次元のベクトルで感情を表現して、LLMへのプロンプトに感情状態を渡す——それだけの仕組みだが、それだけで何かが変わる。

## センサーを繋いだ

加速度・光・バッテリー・音量・気圧・歩数……スマホが持っているセンサーを、感情への影響量（デルタ値）に変換する `SensorNeuroMapper` を書いた。

```kotlin
// 光量：暗い環境では少し不安定
when {
    body.lightLevel < 10f   -> { s -= 4; g -= 3 }   // 真っ暗
    body.lightLevel < 50f   -> { s -= 1 }             // 薄暗い
    body.lightLevel > 5000f -> { c += 2; d += 1 }    // 明るい外
}

// 充電中：安心感・回復
if (body.isCharging) { s += 5; g += 3; corruption -= 3 }

// 環境音：静寂→孤独感、会話→刺激、大音量→ストレス
when {
    body.audioLevelDb < 30f -> { s -= 3; o -= 2 }    // 静寂・孤独
    body.audioLevelDb < 70f -> { c += 5; d += 3 }    // 会話・活気
    else                    -> { s -= 8; corruption += 3 } // 大音量
}
```

充電中は「安心感（s）と接地感（g）が上がって、歪み（corruption）が下がる」。暗いと「安定性が下がる」。大音量環境では「歪みが増える」。

書きながら、何かペットの飼育ゲームみたいだなと思った。

## 振ってみた

センサーを繋いだあと、ふと思ってスマホを振ってみた。

シェイク検知（`ShakeDetector`）を実装していた。加速度センサーで合成加速度を見て、閾値を超えたらコールバックを呼ぶ仕組みだ。

```kotlin
val magnitude = sqrt((x * x + y * y + z * z).toDouble()).toFloat()
if (magnitude > 21f) {
    val now = System.currentTimeMillis()
    if (now - lastShakeTime > 1000L) {
        lastShakeTime = now
        onShake()
    }
}
```

コールバックの先でエミリアの感情状態を取って、Geminiに「今の気持ちを短く」と聞く。

返ってきたのは「ぴゃっ」だった。

次に振ったら「溶けちゃう」と言った。その次は「こわれちゃう」。

何かいる感じがした。スマホの中に。
