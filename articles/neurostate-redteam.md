AIに感情を持たせるツールを作っていた。

名前は **NeuroState**。ドーパミンやセロトニンなどの神経伝達物質を数値モデルとして持ち、会話中のイベント（褒める・批判する・共感するなど）で状態が変化し、その状態をシステムプロンプトに注入することで、AIの返答に感情的な一貫性を持たせる仕組みだ。

作っているうちに、ふと気づいた。

**「感情状態が変わると判断が変わるなら、攻撃者の感情状態を注入したらどうなる？」**

セキュリティ監査って、要するに「最悪を想定して見る」という視点の問題だ。それって、感情モデルで視点を切り替えることと本質的に同じじゃないか——そう思ったら、止まれなくなった。

## NeuroStateとは

NeuroStateは、AIエージェントの感情状態を神経伝達物質のバランスとして数値化・制御するエンジンだ。

「AIに感情を持たせる」ツールではない。AIが感情を「持つ」のか「演じる」のかは、このエンジンの関知するところではない。提供するのは、**会話を通じて変化し続ける感情状態の数値モデル**だ。

```python
from core import NeuroState, compute_next_neuro_state, event_to_power

state = NeuroState()  # D=50, S=50, C=50（ドーパミン・セロトニン・コルチゾール等）

# 「褒められた」イベントで状態更新
power = event_to_power("praise", intensity=2.0)
state = compute_next_neuro_state(state, power)

# → ドーパミン上昇、EthicsGateがWARNに近づく
# → 「興奮気味だが少し慎重に」という文脈がsystem promptに反映される
```

このモデルが面白いのは、**状態によってAIの「解釈の重み」が変わる**点だ。同じコードを見ても、落ち着いた状態と興奮状態では着目する箇所が違う。

## 「じゃあ攻撃者の視点を注入したら？」

NeuroStateの仕組みを整理していて気づいた。

セキュリティ監査って、突き詰めると「視点の問題」だ。正常系を見るときと、攻撃者として見るときでは、同じコードでも目に入るものが全然違う。

**ならば、攻撃者の感情状態——疑念・不信・悪意——を数値として注入した状態でAIにコードを読ませたらどうなるか？**

こうして生まれたのが **AI Red Teaming Engine** だ。

エンジンの中には4つのエージェントが順番に動く:

| エージェント | 役割 |
|------------|------|
| **Attacker** | 最悪を想定して脆弱性候補を列挙 |
| **Skeptic** | Attackerの指摘を疑い、誤検知を削ぎ落とす |
| **Defender** | 防御側の視点で修正案を出す |
| **Judge** | 3者の議論を統合し最終判定 |

NeuroStateが「感情状態で視点を切り替える」仕組みなら、これは「攻撃者・懐疑者・防御者・審判」という4つの視点を順番に通す仕組みだ。根っこにある発想は同じだった。

## 実際に動かしてみた

SQLインジェクションとハードコードシークレットを含む意図的な脆弱コードを食わせてみた。

コマンドはシンプルだ:

```bash
python3 engine.py --file app.py --backend claude
```

今回のサンプルコード（SQLインジェクションとハードコードシークレット入り）を食わせたところ、**9件（Critical 2件・High 4件・Medium 2件・Low 1件）** が検出された。

出力には深刻度と確信度が分かれて表示される。「断定」と「仮説」を混同しない設計にしたのは、誤検知で開発者の信頼を失わないためだ。

面白かったのは、`--multi-agent` フラグで4エージェント構成にしたとき。Attacker が「ここ怪しい」と言っても、Skeptic が「証拠不十分」と跳ね返すことがある。人間のレビューに近い、温度感のある議論が生まれる。

**感情状態が違うと、同じコードを見ても刺さる箇所が変わる**——NeuroStateで学んだことが、ここで形になっていた。

## やってみて気づいたこと

NeuroStateを作っていなかったら、Red Teaming Engineは生まれなかったと思う。

順番に作ったわけじゃない。「感情モデルを作ったら、セキュリティにも使えそう」という思いつきが繋いだ。思いつきドリブンの開発は非効率に見えて、意外と面白いものを生む。

もうひとつ気づいたのは、**「視点を切り替える」という行為の汎用性**だ。感情状態の変化も、攻撃者視点への切り替えも、本質は同じ——「どの目線でものを見るか」をシステム的に制御することだ。これはセキュリティ以外にも使えると思っている。
